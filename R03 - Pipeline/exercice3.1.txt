# Exercice 1: Analyse du pipeline

Pour chacune des lignes de commandes suivantes, décrivez l'objet produit par chaque commande de la chaîne. 
Pour ce faire, analysez l'objet résultant de chaque maillon à l'aide des commandes Get-Member ou Select-Object.


1. "Spooler" | Get-Service | Select-Object * | Format-List | Out-File Spooler.txt

    - "Spooler" (exemple)
        - Type de l'objet en sortie: 
            String
        - Description:
             Le mot "spooler", qui est le nom du service recherché

    - Get-Service
        - Type de l'objet en sortie: 
            Object
        - Description: 
            Créer l'objet en lien avec le service "Spooler"

    - Select-Object
        - Type de l'objet en sortie:
             Détail d'un objet
        - Description:
             Sors le Format-List de l'objet "Spooler"

    - Format-List
        - Type de l'objet en sortie: 
            Liste des "Member" d'un objet
        - Description:
            Liste des "Member" d'un objet

    - Out-File
        - Type de l'objet en sortie:
             Fichier
        - Description: 
            Sors le résultat de la commande suivant le dernier pipeline dans un fichier .txt dans l'emplacement relatif.



2. Get-Item C:\Windows\System32\drivers\etc\hosts | Get-Content | Out-Null

    - Get-Item
        - Type de l'objet en sortie: 
            Paramêtres de l'objet ".\hosts"
        - Description: 
            Créer un objet et affiche les données de l'objet en bref.

   - Get-Content
        - Type de l'objet en sortie: 
            String
        - Description: 
        Affiche le contenu du fichier C:\Windows\System32\drivers\etc\hosts dans la console comme un string

    - Out-Null
        - Type de l'objet en sortie: 
            Null
        - Description: 
            Retourne dans la console powershell le néant
